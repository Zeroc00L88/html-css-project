Le fonctionnement Client - Serveur

Lorsque vous voulez visiter un site web, vous tapez son adresse dans votre navigateur web. Mais ne vous êtes-vous jamais demandé comment faisait la page web pour arriver jusqu'à vous ?

Il faut savoir qu'Internet est un réseau composé d'ordinateurs. Ceux-ci peuvent être classés en deux catégories :

  1. Les clients : ce sont les ordinateurs des internautes comme vous. Votre ordinateur fait donc partie de la catégorie des clients. Chaque client représente un visiteur d'un site web.
  2. Les serveurs : ce sont des ordinateurs puissants qui stockent et délivrent des sites web aux internautes, c'est-à-dire aux clients. La plupart des internautes n'ont jamais vu un serveur de leur vie. Pourtant, les serveurs sont indispensables au bon fonctionnement du Web.

Coté client

Javascript est utilisé pour rendre dynamiques les pages que nous visitons, déclencher une animation lors d'un clique, gérer et animer les éléments de votre site, inclure et gérer des composants audios et videos et plus encore

Coté serveur

Si les fonctionnalités de javascript sur le navigateur web permettent de nombreuses choses, il manque une partie essentielle de nos applications web modernes qui réside dans la possibilité de mettre à jour de manière dynamique le contenu d'un site en se basant sur des données elles aussi dynamiques stockées sur un serveur.

Le php viens compléter le javascript coté serveur afin de permettre les echanges entre l'utilisateur et les données hébergées par l'application web.

Si le php est trés connu pour la gestion coté serveur le javascript ainsi que d'autres languages de programmation peuvent aujourd'hui être utilisés pour programmer la partie serveur
